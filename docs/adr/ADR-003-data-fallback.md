# ADR-003: 3단계 데이터 폴백 캐스케이드 채택

- **날짜:** 2026-03-20
- **상태:** 승인됨

## 결정

외부 API 데이터 수집 실패 시 `실시간 → DB 캐시 → 추정값` 3단계 폴백을 모든 API 엔드포인트에 적용한다.

## 배경

- pykrx(KRX API)는 장중/장 종료 후 응답 불안정 사례가 빈번
- DART API는 월별 할당량 제한 존재 (분당 요청 제한)
- KRX API 컬럼명이 업데이트 공지 없이 변경되는 사례 존재
- 투자 분석 앱에서 "데이터 없음" 화면 표시는 사용자 신뢰 손상

## 선택된 방안

**3단계 캐스케이드:**
1. **실시간 (Primary):** pykrx / DART API 직접 호출 (5초 타임아웃)
2. **캐시 (Secondary):** `api_cache` 테이블에서 최근 저장 데이터 반환
3. **추정 (Tertiary):** 재무제표 기반 역산 추정 (`EPS × 섹터PER`, `영업이익 × 1.2`)

## 대안과 기각 이유

| 대안 | 기각 이유 |
|------|----------|
| 단순 실패 처리 | 사용자에게 "오류" 화면 빈번하게 노출 |
| 실시간만 사용 | KRX 불안정 시 전체 기능 마비 |
| 캐시만 사용 | 장 중 실시간 데이터 제공 불가 |

## 결과

- 모든 시장 데이터 API 응답에 `"fallback": bool` 메타 필드 포함
- 프론트엔드는 `fallback=true` 시 노란색 경고 배너 표시
- FCF 추정 로직: `op_cash_flow == 0` → `fcf = operating_income × 1.2`
- 이 결정 없이는 KRX API 불안정 시 스크리너/대시보드 전체 마비
