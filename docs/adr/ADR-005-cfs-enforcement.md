# ADR-005: DART 연결재무제표(CFS) 강제 채택

- **날짜:** 2026-03-20
- **상태:** 승인됨

## 결정

DART API 호출 시 `fs_div="CFS"` (연결재무제표)를 항상 강제한다. 별도재무제표(`OFS`) 사용을 코드레벨에서 차단한다.

## 배경

- 삼성전자, SK하이닉스 등 대형 지주·계열사 구조 기업은 별도재무제표 매출이 연결 대비 수십% 낮음
- 투자자 관점의 실질 수익성은 연결 기준이 정확
- 일부 증권 앱은 별도재무제표 기반 지표를 제공하여 투자 판단 왜곡 초래
- Finview의 핵심 가치인 "왜곡 없는 재무 데이터 제공"에 직결

## 선택된 방안

`core/dart_api.py`의 `get_financial_statement()` 기본 파라미터에 `fs_div="CFS"` 하드코딩:

```python
def get_financial_statement(self, corp_code, bsns_year, reprt_code="11011"):
    params = {
        ...,
        "fs_div": "CFS",  # 변경 불가 — ADR-005
    }
```

## 대안과 기각 이유

| 대안 | 기각 이유 |
|------|----------|
| OFS를 옵션으로 제공 | 사용자 혼란 및 투자 판단 오류 위험 |
| 사용자가 선택 | 대부분 사용자가 차이를 모름 |
| 자동 감지 | 종목마다 CFS 제출 여부가 다름 → 일관성 깨짐 |

## 결과

- 모든 재무 KPI (ROE, op_margin, debt_ratio 등)가 연결 기준으로 일관되게 산출
- 소규모 기업(연결 없이 별도만 제출)의 경우 DART API 자체에서 CFS 반환 불가 → `빈 데이터` 처리
- 이 결정은 데이터 정확성을 위한 intentional 트레이드오프
